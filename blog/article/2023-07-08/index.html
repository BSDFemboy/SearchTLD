Language dropdowns. Be gone, .php!

Now instead of having to type your language's ISO693_1 code, you can just select your language from a drop-down list. This is done using the restcountries API.


This next one is for people who host SearchTLD, now in config.json you can enable and disable the .php extension from URLs.

Note: To actually be able to remove the .php extension you need to allow URL Rewriting In your NGINX / Apache config. If you use NGINX, rewriting is automatically enabled in the updated sample configurations.
